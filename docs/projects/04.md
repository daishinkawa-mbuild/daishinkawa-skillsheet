# 医療系DXシステムの開発

## 期間
2025年2月〜現在（1年1ヶ月）

## プロジェクト概要
オンプレミス環境で稼働する医療系DXシステムの開発に従事。Django REST Framework と Next.js を用いたWebアプリケーションのフロントエンド・バックエンド双方を担当。

少人数チーム（4名）において、機能実装に加え、監査要件を満たす機能設計やテスト基盤整備など、品質・保守性を意識した開発を推進。

現在はMVPから製品化に伴い、バックエンドをNestJSに刷新する対応を、技術選定、設計を含め一気通貫で推進中。

---

## 役割・担当領域
- フルスタックエンジニアとしてフロントエンド・バックエンドの設計・実装を担当
- 監査系機能における要件整理・設計・実装を主導
- E2Eテスト基盤の設計・構築およびテストコード実装
- 小規模チームにおける設計・実装の推進

---

## 実績・取り組み

### 監査要件を考慮した機能設計
- 医療系システム特有の監査要件に対応するため、操作履歴や変更履歴の管理機能を設計・実装
- データの追跡性・再現性を意識した設計を行い、監査対応可能な構造を構築

### E2Eテストの導入と品質向上
- Playwrightを用いたE2Eテスト環境を構築
- 主要機能のシナリオテストを自動化し、リグレッション検知を仕組み化
- GitHub Actionsと連携し、CI上での自動テスト実行フローを整備

### フロントエンド・バックエンドの一貫した実装
- Next.js（TypeScript）によるフロントエンド実装と、Django REST FrameworkによるAPI設計・実装を担当
- フロントとバックエンドの責務分離やデータ構造設計を意識し、変更に強い構成を実現

---

## 技術スタック
- フロントエンド: React / Next.js / TypeScript
- バックエンド: Python / Django REST Framework
- テスト: Playwright
- インフラ / 環境: Docker / Docker Compose（オンプレミス環境）
- CI/CD: GitHub Actions
- 開発支援: codex、Devin
